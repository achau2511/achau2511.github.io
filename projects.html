Pairs Trading & Statistical Arbitrage Dashboard

Python yfinance statsmodels Plotly Panel NumPy Pandas
Interactive dashboard showing equity curve for KO vs PEP pair
View on GitHub →

This project explores statistical arbitrage through pairs trading, a market-neutral strategy that profits from the temporary divergence of two historically correlated assets. The core idea is that if two stocks tend to move together, a significant deviation from their typical relationship represents a potential trading opportunity.

I built an interactive dashboard using Panel and Plotly that allows users to input any two equity tickers and automatically runs a suite of statistical tests to evaluate the pair. This includes cointegration testing via the Engle-Granger method, spread calculation, and rolling beta estimation to capture evolving relationships between assets over time.

The dashboard computes key metrics including the Sharpe ratio, p-values for statistical significance, and mean-reversion speed, giving users a quantitative basis for evaluating whether a pair is worth trading. Users can also customize the backtesting date range to test how strategies would have performed across different market regimes.

  • Implemented cointegration testing and spread analysis to identify mean-reverting equity pairs
  • Built long-short backtesting engine with customizable entry/exit thresholds based on z-score signals
  • Visualized equity curves, rolling correlations, and spread dynamics using interactive Plotly charts
Dec 2025 – Jan 2026 · NLP / ML

Email Classification with Transformer-Based NLP

Python PyTorch Hugging Face Pandas NumPy scikit-learn

Email overload is a universal problem, and this project tackles it by building a multi-class email classifier that automatically categorizes messages as spam, promotions, or primary inbox. The project was an opportunity to work hands-on with transformer architecture and understand how modern NLP models process and represent text.

The pipeline begins with raw email text, which is passed through a tokenizer from Hugging Face to convert it into token IDs and attention masks. These are then fed into a pre-trained transformer model fine-tuned on the classification task. I implemented the full training loop in PyTorch, including batching, gradient updates, and learning rate scheduling.

Model performance was evaluated using precision, recall, and F1-score across all three categories, which provided a more nuanced view of performance than accuracy alone — particularly important given the class imbalance typical in email datasets.

  • Fine-tuned a pre-trained transformer model on a dataset of 1K+ labeled emails
  • Built a full tokenization and embedding pipeline from raw text to model-ready tensors
  • Achieved strong F1 performance across all three categories, with highest precision on spam detection
Oct – Dec 2025 · Finance / ML

Stock Movement Prediction

Python yfinance scikit-learn TensorFlow Pandas NumPy Matplotlib

Predicting short-term stock price movements is one of the most challenging problems in quantitative finance. This project approached it as a binary classification problem — given historical price and volume data, can a model predict whether a stock will go up or down the next day?

I pulled 5K records of historical market data using the yfinance library across multiple sectors, then engineered features including lagged returns, rolling averages, and volume indicators. Two models were trained and compared: a logistic regression baseline and an LSTM neural network designed to capture temporal dependencies in sequential price data.

The LSTM achieved 58% accuracy on held-out test data, modestly outperforming the logistic regression baseline. The project also included a sector-level analysis comparing model performance across tech, energy, and financial stocks, revealing meaningful differences in predictability across industries.

  • Engineered time-series features including lagged returns, rolling means, and volume ratios
  • Compared LSTM and logistic regression performance across multiple equity sectors
  • Analyzed prediction confidence and error patterns to identify where models struggled most
Aug – Oct 2025 · Data Analysis / ML

Music Popularity Prediction

Python Pandas NumPy scikit-learn Matplotlib Seaborn Spotify API

What makes a song popular? This project used Spotify's audio feature data to answer that question through exploratory data analysis, clustering, and regression modeling. Spotify provides rich acoustic metadata for every track — including danceability, energy, valence, tempo, and loudness — making it an ideal dataset for this kind of analysis.

I applied K-Means clustering to group tracks by their acoustic profiles, revealing distinct musical archetypes — high-energy dance tracks, mellow acoustic songs, and mid-tempo pop. These clusters were then visualized using dimensionality reduction to make the groupings interpretable.

A regression model was trained to predict a track's popularity score based on its audio features. The analysis found that danceability, energy, and valence were the strongest predictors of popularity, while features like acousticness and instrumentalness were negatively correlated with chart performance.

  • Performed EDA on Spotify track data to identify features most correlated with popularity
  • Applied K-Means clustering to discover distinct acoustic archetypes within the dataset
  • Built and evaluated a regression model to quantify the relationship between audio features and popularity
